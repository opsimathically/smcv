# SMCV product design guidelines

These documents govern all owner-facing web and CLI experiences:

- [Product design system](PRODUCT_DESIGN_SYSTEM.md)
- [Content and trust language](CONTENT_AND_TRUST_LANGUAGE.md)
- [Accessibility standard](ACCESSIBILITY.md)
- [Critical user flows](CRITICAL_USER_FLOWS.md)

The intended character is calm, precise, restrained, and trustworthy. SMCV is
not styled as a hacker terminal, cryptocurrency product, generic cloud
dashboard, or fear-driven security warning system.

Design decisions must make current state, effective authority, data exposure,
irreversibility, and recovery consequences understandable before action.
